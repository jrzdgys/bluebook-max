# 蓝宝书Max

> 基于 Alpha派 蓝宝书的机构级投研情报工具。全链路自动化：数据提取 → 分类评分 → 摘要精炼 → HTML生成 → GitHub Pages部署。

---

## 二、HTML 模板架构（冻结）

### 2.1 CSS 核心规范

| 组件 | 类名 | 说明 |
|------|------|------|
| 页面容器 | `.w` | max-width:900px, 居中 |
| 头部 | `.sh` | 深蓝渐变背景，圆角顶部 |
| 内容区 | `.ct` | 米白背景，圆角底部 |
| 市场摘要 | `.ms` | 灰色底，13px |
| 主题卡片 | `.tc` | 白底圆角卡片，hover蓝色边框 |
| 阶段标签 | `.tc-stage` | 小圆角标签，颜色由JS动态设置 |
| 热度指数 | `.tc-heat` | 20px 加粗，可点击弹出评分详情 |
| 评分弹窗 | `.hp` | position:fixed，深色背景，智能边缘检测 |
| 股票行 | `.sr` | flex布局：名称→代码→价格→涨跌幅→理由 |
| 价格 | `.sp` | 12px desktop, 11px mobile |
| 涨跌幅 | `.spp` | 与价格并列 |
| 分组标签 | `.gl.l1/.l2/.l3` | 龙头首选(橙)/弹性机会(蓝紫)/相关标的(灰) |

### 2.2 移动端适配

```css
@media (max-width: 640px) {
  .sr2 { flex-basis: 100%; white-space: normal; overflow: visible; }
  body { padding: 12px; }
  .sh { padding: 16px 18px 14px; }
  .ct { padding: 14px 16px 20px; }
}
```

### 2.3 JS 核心函数

| 函数 | 用途 |
|------|------|
| `ht(h)` | 热度→颜色：>=85红, >=65橙, >=50蓝, <50灰 |
| `sb(h)` | 热度→阶段背景色（带透明度） |
| `isTH()` | 判断是否交易时段（周一至五 9:30-11:30, 13:00-15:00 CST） |
| `fq()` | 东方财富实时行情轮询（盘中15s/盘后60s） |
| `sp(idx,el)` | 评分弹窗：fixed定位 + 智能边缘检测 + 3维度明细 |

### 2.4 实时行情架构

- **API**: `https://push2.eastmoney.com/api/qt/ulist.np/get`
- **参数**: `fltt=2&invt=2&fields=f2,f3,f12,f14&secids=...`
- **secid格式**: `1.XXXXXX`（沪市）, `0.XXXXXX`（深市）
- **轮询间隔**: 盘中15秒 / 盘后60秒
- **颜色**: 红涨绿跌（>=0红色 .up, <0绿色 .dn），涨跌幅保留2位小数

### 3.3 secidMap

```json

```

- 前缀 `0.` = 深市（000/001/002/300/301）
- 前缀 `1.` = 沪市（600/601/603/605/688）
- 仅包含有SECID代码的A股

## 四、标的分类规则（决策树 v2.1）

分类由 Alpha派原文的组序和角色定位驱动。

| 规则 | 条件 | 分类 |
|------|------|------|
| R0 | 市值 < 50亿 | 不能为龙头首选（可信度底线） |
| R1 | 原文第一组首位 + 市值 >= 50亿 | 龙头首选 |
| R2 | 原文第一组（非首位） | 弹性机会 |
| R3 | 原文第二组 + 角色含"核心/直接/龙头/充分" | 弹性机会 |
| R4 | 市值 > 300亿 + 主题内排名前40% | 弹性机会 |
| R5 | 其余所有 | 相关标的 |

**简化版（用于快速生成）：**
- 1只股票 → 全部归入"龙头首选"
- 2-5只 → 第1只"龙头首选"，2-3只"弹性机会"，其余"相关标的"
- 6+只 → 第1只"龙头首选"，2-4只"弹性机会"，其余"相关标的"

## 五、主题评分规则（三层独立评分 v2）

```
热度总分(0-100) = 机构关注度(0-60) + 市场确认度(0-25) + 催化强度(0-15)
```

### 5.1 机构关注度（0-60分）

基于Alpha派主题排名和机构覆盖密度综合评定。

### 5.2 市场确认度（0-25分）

基于该主题全部标的的平均涨跌幅绝对值。

| 均价涨幅 | 得分 |
|----------|------|
| >=5% | 25 |
| >=3% | 18 |
| >=1% | 10 |
| <1% 或无数据 | 5 |

### 5.3 催化强度（0-15分）

基于Alpha派原文关键词匹配：

| 催化剂类型 | 得分 | 关键词 |
|------------|------|--------|
| 确定性催化剂 | 15 | 官宣、确认、正式发布、业绩、获批、正式、量产 |
| 高置信度预期 | 10 | 或将、预计、接近、计划、推进、验证 |
| 传闻/跟踪 | 8 | 传、据称、关注、跟踪、预期 |

### 5.4 阶段判定（阈值：85/65/50）

| 总分 | 阶段 | 颜色 | 含义 |
|------|------|------|------|
| >=85 | 🔥 主升 | #C4433A 红 | 机构高度聚焦，催化剂已触发 |
| >=65 | 📈 强化 | #E67E22 橙 | 新催化加入，关注度显著上升 |
| >=50 | ➡️ 持续 | #2E86C1 蓝 | 维持关注，平稳运行 |
| <50 | 🌱 孵化 | #7D8B8F 灰 | 早期信号，待催化确认 |

## 六、摘要与引用自动提取规则

从Alpha派原始摘要中全自动提取，不依赖历史版本缓存。

### 6.1 摘要提取（R1-R5）

| 规则 | 操作 | 示例 |
|------|------|------|
| R1 | 去除时间/背景引导语 | "在过去24小时内"、"6月22日"、"美国当地时间"、"截至…"、"昨晚(…)" |
| R2 | 去除元论述过渡句 | "这一事件的边际变化在于"、"这标志着"、"此举意味着" |
| R3 | 保留核心事件（<=85字） | 取前1-2句事实性陈述 |
| R4 | 去除Markdown标记 | 移除所有 **...** |
| R5 | 排除"关注："段 | 股票列表不在摘要中出现 |

### 6.2 引用提取（Q1-Q4）

| 规则 | 操作 |
|------|------|
| Q1 | 去除元论述引导语（同R1/R2） |
| Q2 | 去除Markdown标记 |
| Q3 | 优先高冲击力论点（对比结构 > 动词驱动 > 关键词） |
| Q4 | 控制在60字以内 |

## 七、股票推荐理由提取规则

**照抄Alpha派原文"关注："段，不做任何修改。**

Alpha派原文格式：
```
关注：**股票A**/**股票B**（推荐理由），**股票C**（推荐理由2）
```

- `/` 分隔的同组股票共享同一括号内的理由
- 每个主题独立提取
- 如原文未提供理由，使用所属分类标签作为fallback
