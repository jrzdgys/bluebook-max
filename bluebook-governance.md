# 蓝宝书Max 项目治理规则

> 本文件定义了蓝宝书Max项目的核心约束和开发流程。
> **每次修改前必须重新阅读本文件**，确保变更不会破坏冻结区和约束区的规则。

---

## 🔴 一、冻结区（未经明确授权不得触碰）

以下内容是项目的基石，任何修改必须由用户**明确指示**，不得自行调整：

### 1.1 报告模板结构（template-v3.html）
- 报告卡片、主题、标的的DOM层级结构
- 股价涨跌幅的**红涨绿跌**配色规则
- 非A股标的：**只显示名称+推荐理由**，不显示股价栏、不留占位符
- 重合标的卡片：**仅含A股**（通过 secidMap 的 startsWith('0.') || startsWith('1.') 过滤）
- 各板块的顺序：今日聚焦 → 重合标的 → 主题卡片 → 机会前瞻

### 1.2 数据规则
- **数据源仅使用东方财富 API**（push2.eastmoney.com），弃用腾讯API
- A股判定：只能通过 `searchadapter.eastmoney.com` 的 MktNum 字段（0/1为A股）
- A股匹配：先精准匹配 → 失败则去除 -U/-W/-V 等后缀再匹配

### 1.3 激活码系统
- 激活码格式：`BBM-8位随机字母数字组合`
- 最大绑定设备数：2
- 本地降级验证：`codes.js` 中的 `VALID_HASHES`（SHA-256）
- token有效期：至少30天（`Math.max(sub.expiresAt, Date.now() + 30天)`）

### 1.4 文件命名规范
- 报告文件：`{版型代码}-{YYYYMMDD}.html`（mc=晨会版, pm=午间版, ev=晚间版, gv=全球版）
- 导航页：`index.html`
- 测试版（仅当商讨升级/迭代模板时使用）：文件名加 `-test` 后缀
- 日常报告生成直接出正式版，不走测试版流程

### 1.5 Worker 鉴权逻辑
- expiresAt 类型安全转换（ISO字符串 → 时间戳）
- isAuthenticated() 遇到 NaN expiry 时自动清除坏token
- 不得修改鉴权的核心验证链

## 🟡 二、约束区（修改需遵循的规范）

### 2.1 报告生成流程
```
ED JSON → 注入 template-v3.html → 替换静态 title/og:title → 推送
```
- 必须先生成 ED JSON，再注入模板
- 替换静态 title/og:title（爬虫不执行JS）
- 不改 manifest.json 的 entries / reports 双key设计
- 不改导航页的 getWeekInfo、fmtRange、getPrevDate 等日期函数行为

### 2.2 导航页分组规则（index.html）
- 最新报告 → "今天" 分组
- 本周 → 默认展开（▾, max-height:6000px）
- 上周 → 默认折叠（▸, data-c="1", max-height:0px）
- 更早 → 默认折叠，内按周分子组

### 2.3 数据获取
- 可复用历史库已获取的数据，不重复请求 Alpha派
- 价格数据仅用东方财富 push2 API
- 指数数据来自 push2.eastmoney.com

### 2.4 HTML/JS 操作规范
- JS 字符串拼接中的引号转义注意：`\'` 在 JS 单引号字符串内是转义的单引号，在表达式上下文是语法错误
- 若需条件渲染 HTML 属性，使用**变量预定义**方式而非内联三元（避免转义问题）

### 历史踩坑记录
| 问题 | 原因 | 修复 |
|------|------|------|
| Worker token 过期 NaN | expiresAt 存 ISO 字符串，Math.max(string, number) => NaN | 加类型安全转换 |
| 非A股显示股价栏 | 未过滤 secidMap 中不存在或非A股的标的 | 添加 startsWith 过滤 |
| CDN 缓存导致旧代码 | GitHub Pages CDN 约 2-5 分钟生效 | 刷新后加 `?v=N` 参数 |
| Worker 部署 10021 错误 | Content-Type 用错 | 改用 application/javascript+module |
| secid 前缀错误 | 润建股份(002929)和亚康股份(301085)使用1.前缀但API需0.前缀 | 使用0.前缀统一 |
